## Interpretation Rules

This report supports only measured claims. On TinyStories, a sparse run is not a same-quality efficiency win unless it remains close to Dense eval loss while improving a named efficiency axis such as VRAM, trainable parameters, checkpoint size, cached-tail training time, or active expert compute. If generated-code evaluation is enabled, verifier quality should also be checked.

## Limitations

- One Colab run is not a paper-grade result.
- TinyStories is a small story-style language-modeling corpus, not a coding benchmark.
- L4 availability, thermal behavior, and Colab scheduling can change throughput.
- Short `MAX_STEPS` values are useful for workflow validation but not final quality claims.
- CPU fallback results must not be interpreted as GPU efficiency evidence.
